# HelmCode 后续增强 backlog

> 不是 v1.0 阻塞项,只记录"未来可能扩展的目录类别 / 模板",供技能维护者参考。

## init-java-ddd

### v1.1 候选:可选目录占位

参考工程 `mycmbillmanage` 比 `init-java-ddd` 生成结果多出以下目录类别。
**全部是后续业务才需要的**,保持 v1.0 demo 切片不引入这些类别 ✅。
若未来需要扩展,可考虑在 SKILL.md 增加 `withScheduler` / `withMq` / `withIntegration` flag:

| 模块 | 缺失类别 | 何时需要 |
|---|---|---|
| application | `controller/` | 用户加 REST API(目前 web 模块独立) |
| application | `scheduler/<biz>/` | 加定时任务/批处理 |
| application | `service/<biz>/{impl,command,vo}` | 业务复杂度上来后细分 |
| facade | `<biz>/{command,request,vo,enums}/` | 加新业务切片 |
| facade | `common/{request,vo}/` | 沉淀跨切片通用 DTO |
| infrastructure | `integration/<rpc-target>/{request,response,impl}/` | 接外部 RPC |
| infrastructure | `event/`、`mq/` | 接消息队列 |
| infrastructure | `drm/`、`zcache/` | 接 DRM 配置中心、Tair 缓存 |
| domain | `model/<biz>/`、`service/impl/<biz>/` | 加新业务切片 |
| domain | `query/<biz>/`、`command/<biz>/` | CQRS 化 |

### 不打算加的

- `application/controller/*` — 我们用独立 web 模块,不混进 application
- `service/llm/`、`service/aibillparse/` — 业务专属,不通用
